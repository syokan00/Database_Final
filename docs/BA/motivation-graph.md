# モチベーショングラフ（Motivation Graph）

本ドキュメントでは、代表的なユーザー（ペルソナ 1：山田 太郎）が  
MemoLucky を利用するまでの「感情・動機の変化」をテキストベースで整理する。

## 1. 時系列の流れ

### ステージ 1：現状の不満（Before）

- 状況：
  - 課題の締切、バイトのシフト、買い物メモなどがバラバラの場所に保存されている
  - 不要な本や小物を処分したいが、特に行動には移していない
- 感情：
  - 「どこにメモしたか分からなくなるのがストレス」
  - 「部屋が少しずつ物で溢れてきている」
- モチベーション：
  - **低い〜普通**  
  - まだ「何かツールを探そう」とは思っていない状態

---

### ステージ 2：課題の増加による危機感

- 状況：
  - 学期末が近づき、複数の授業でレポートや最終課題が重なっている
  - どの課題がいつまでなのかを整理したい
- 感情：
  - 「このままだと締切を忘れそうで不安」
  - 「一度、ちゃんと整理する仕組みを作りたい」
- モチベーション：
  - **上昇**  
  - 「管理ツールが必要かもしれない」と感じ始める

---

### ステージ 3：解決策の探索

- 状況：
  - 既存のメモアプリ／タスク管理アプリをいくつか試してみる
  - フリマアプリも調べるが、「メモ」と「出品」を一緒に管理するものはあまりない
- 感情：
  - 「メモ専用アプリはあるが、物の出品まではできない」
  - 「フリマアプリは便利だが、授業の道具などをクラスメイトに譲るには少し大げさ」
- モチベーション：
  - **中〜高**  
  - 「自分の用途に合う、もっとシンプルなツールが欲しい」と強く感じ始める

---

### ステージ 4：MemoLucky との出会い

- 状況：
  - 授業や友人を通じて、MemoLucky の存在を知る
  - GitHub Pages の URL を受け取り、ブラウザでアクセスしてみる
- 感情：
  - 「ブラウザだけで動くなら、インストールもいらないし試しやすい」
  - 「メモと出品が同じ UI で管理できるのは便利そう」
- モチベーション：
  - **高い**  
  - 「一度ちゃんと使ってみよう」と行動に移る

---

### ステージ 5：初回利用

- 状況：
  - メモ一覧画面で、サンプルのメモや自分のメモを確認
  - 出品一覧画面で、参考書などの出品イメージをつかむ
- 感情：
  - 「UI がシンプルで分かりやすい」
  - 「これなら授業用のメモや出品管理に使えそう」
- モチベーション：
  - **ピークに達する**  
  - 「今学期の課題管理と不要物の整理に使ってみよう」と決意する

---

### ステージ 6：継続利用（After）

- 状況：
  - 課題やタスクを MemoLucky 上で整理するのが習慣化
  - 不要な本や道具も、このアプリで出品してみる
- 感情：
  - 「メモと出品を一緒に見られるので、頭の中もすっきりする」
  - 「同じ学生同士で物をやり取りできる場としても便利」
- モチベーション：
  - **安定して高い**  
  - 「今後の学期や他のプロジェクトでも使い続けたい」と感じる

---

## 2. グラフ化イメージ（テキスト）

- 横軸：時間（Before → After）  
- 縦軸：モチベーション・満足度  

おおまかな推移：

1. 現状の不満：低〜普通  
2. 課題の増加：少し上昇  
3. 解決策の探索：中〜高  
4. MemoLucky を発見：高  
5. 初回利用：ピーク  
6. 継続利用：やや落ち着きつつも、高い水準で安定  

この説明をもとに、プレゼン資料やレポートでは  
折れ線グラフとして可視化することができる。
