# UI モック（UI Mock）仕様書

本ドキュメントでは、MemoLucky の主要画面について、  
UI 構成要素をテキストベースで定義する。  
実際の提出時には、本内容をもとにワイヤーフレームを手書き・ツールで作成し、画像として提出できる。

---

## 1. 共通レイアウト

### 1.1 ヘッダー（ナビゲーションバー）

- 要素：
  - 左側：アプリ名ロゴ「MemoLucky」
  - 中央〜右側：ナビゲーションリンク
    - 「メモ」（メモ一覧ページへのリンク）
    - 「出品」（マーケット / 出品一覧ページへのリンク）
    - 「プロフィール」（ユーザー情報ページへのリンク）
  - 右端（オプション）：
    - 「ログイン / ログアウト」ボタン（設計上）

- 挙動：
  - すべてのページで共通して表示される
  - 現在のページに応じて、対応するメニューを強調表示する（太字・下線など）

---

## 2. メモ一覧画面（Notes）

### 2.1 画面構成

- 上部：
  - ページタイトル「メモ一覧」
  - 右上に「新規メモ」ボタン（設計）

- 中央：
  - メモカードのグリッドまたはリスト表示
  - 各カードの要素：
    - タイトル
    - 内容の抜粋（先頭数行）
    - 作成日または更新日
    - （オプション）タグ

- 下部：
  - ページネーションまたは「もっと見る」ボタン（必要に応じて）

### 2.2 ユーザー操作

- カードをクリックすると、メモの詳細をポップアップまたは別ページで表示（設計）
- 「新規メモ」ボタンから、タイトル・内容を入力して保存できる（設計）

---

## 3. 出品一覧画面（Items / Marketplace）

### 3.1 画面構成

- 上部：
  - ページタイトル「出品一覧」
  - 右上に「新規出品」ボタン（ログインユーザー向け、設計）
  - 絞り込みエリア（オプション）：
    - キーワード検索欄
    - ステータス（募集中 / 交渉中 など）のフィルタ

- 中央：
  - 出品アイテムカードのリスト（縦方向）
  - 各カードの要素：
    - タイトル
    - 価格
    - ステータス（日本語ラベル：「募集中」「交渉中」など）
    - 短い説明文の抜粋
    - タグ一覧（あれば）
    - 「詳細」ボタン
    - （所有者の場合のみ）「編集」「削除」ボタン

### 3.2 ユーザー操作

- 「詳細」ボタン：
  - 別ページまたはモーダルで、アイテムの詳細情報を表示
- 「編集」ボタン（所有者のみ）：
  - アイテム編集ページへ遷移
- 「削除」ボタン（所有者のみ）：
  - 確認ダイアログを表示し、OK の場合は削除処理を実行（設計）

---

## 4. 出品編集 / 新規出品画面

### 4.1 入力フォーム項目

- タイトル（必須）
- 説明（任意、複数行テキスト）
- 価格（数値、ステップは 1 単位）
- ステータス（選択式：募集中 / 交渉中 など）
- タグ（カンマ区切り、または簡易入力）

### 4.2 ボタン

- 「保存」ボタン：
  - 新規作成または更新処理を実行
- 「キャンセル」ボタン：
  - 前の画面（一覧など）に戻る

---

## 5. プロフィール画面

### 5.1 表示項目

- ユーザー名
- メールアドレス
- アカウント作成日（オプション）

### 5.2 操作項目

- パスワード変更フォーム（現パスワード／新パスワード／確認用）
- （オプション）自分の出品一覧へのリンク

---

## 6. UI モック作成のポイント

- ワイヤーフレームは「白黒ベース」で構わない  
  - コンポーネント（カード、ボタン、入力欄）の配置が分かるように描く  
- 本ドキュメントの各画面構成をもとに、  
  - ページタイトル
  - 主なボタン
  - カードの情報量  
 などを図示すれば、授業提出用の UI モックとして十分な内容になる。
