# GitHub Kanban Project タスク一覧

このドキュメントには、GitHub Kanban Project に追加するタスクカードの内容が記載されています。各タスクを GitHub Project の看板に追加する際は、この内容をコピーして使用してください。

## 列構成

- **Backlog**: 今後実装予定のタスク
- **Ready**: 実装準備が整ったタスク
- **In Progress**: 現在作業中のタスク
- **In Review**: レビュー中のタスク
- **Done**: 完了したタスク

## タスクの追加方法

1. GitHub リポジトリの **Projects** タブにアクセス
2. Kanban プロジェクトを開く
3. **Add item** ボタンをクリック
4. タスクのタイトルを入力（上記のタスク名をコピー）
5. タスクの説明を入力（上記の説明をコピー）
6. ラベルを追加（関連ロール: `BA`, `Architect`, `DBA`, `Infra`, `Application`, `PM`）
7. 適切な列に配置（Backlog / Ready / In Progress / In Review / Done）

## タスクの管理

- **優先度**: 高 / 中 / 低
- **見積もり**: 時間単位
- **関連ロール**: BA / Architect / DBA / Infra / Application / PM
- **ステータス**: Backlog / Ready / In Progress / In Review / Done
