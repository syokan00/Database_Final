# アプリケーション概要

## システムの目的

**MemoLucky** は、大学生向けのキャンパスライフ支援プラットフォームです。先輩たちのリアルな経験談や情報を共有し、学生同士が助け合えるコミュニティを構築することを目的としています。

### スローガン
**"Lucky for you — あなたに届く、誰かの経験。"**

## ターゲットユーザー

- **大学生**: 特に新入生や下級生
- **大学院生**: 研究室選びや就活情報を探している学生
- **卒業生**: 後輩に経験をシェアしたい人

## 主要機能一覧

### 1. ユーザー管理
- **新規登録**: メールアドレスとパスワードでアカウント作成
- **ログイン/ログアウト**: JWT トークンベースの認証
- **プロフィール管理**: ユーザー情報の表示と編集

### 2. 経験談（Notes）
- **投稿作成**: タイトル、本文、タグ、カテゴリで経験談を投稿
- **投稿一覧**: 時系列順または人気順で投稿を表示
- **投稿詳細**: 個別の投稿内容を閲覧
- **いいね機能**: 投稿に「いいね」を付ける
- **検索機能**: キーワードやタグで投稿を検索

### 3. フリマ（Items）
- **出品**: タイトル、説明、価格、画像でアイテムを出品
- **出品一覧**: カテゴリやステータスでフィルタリング
- **出品編集**: 自分の出品アイテムを編集
- **出品削除**: 自分の出品アイテムを削除
- **価格設定**: 任意の価格を設定可能（100 円単位の制限なし）

### 4. 研究室情報（Labs）
- **研究室情報の閲覧**: 各研究室の情報や先輩の体験談を確認

### 5. 就活情報（Jobs）
- **就活体験談の閲覧**: 先輩の就活体験談や選考情報を確認

### 6. バッジ（Badges）
- **バッジシステム**: 投稿数や活動に応じたバッジ獲得（開発中）

### 7. チャット（Chat）
- **メッセージング**: ユーザー間のメッセージ交換（開発中）

## 画面一覧

### 1. ホーム画面（HomePage）
- **URL**: `/`
- **説明**: アプリのトップページ
- **主要要素**:
  - ヒーローセクション（アプリの紹介）
  - 新着投稿一覧
  - 人気投稿一覧
  - 「今すぐ始める」ボタン

### 2. 経験談一覧画面（NotesPage）
- **URL**: `/notes`
- **説明**: すべての経験談投稿を一覧表示
- **主要要素**:
  - 投稿カード一覧
  - 検索バー
  - フィルタリング機能（カテゴリ、タグ）

### 3. フリマ一覧画面（ItemsPage）
- **URL**: `/items`
- **説明**: 出品アイテムを一覧表示
- **主要要素**:
  - アイテムカード一覧（画像、タイトル、価格、ステータス）
  - 検索・フィルタリング機能
  - 「出品する」ボタン

### 4. 投稿作成画面（CreatePost）
- **URL**: `/create`
- **説明**: 新しい投稿（経験談または出品）を作成
- **主要要素**:
  - 投稿タイプ選択（経験談 / 出品）
  - タイトル入力欄
  - 本文入力欄
  - タグ入力欄
  - カテゴリ選択
  - 画像アップロード（出品の場合）
  - 価格入力（出品の場合）
  - 「保存」ボタン

### 5. 出品編集画面（EditItemPage）
- **URL**: `/items/:id/edit`
- **説明**: 自分の出品アイテムを編集
- **主要要素**:
  - 既存データの表示（タイトル、説明、価格など）
  - 編集フォーム
  - 「保存」ボタン
  - 「削除」ボタン

### 6. 研究室画面（LabsPage）
- **URL**: `/labs`
- **説明**: 研究室情報の一覧と詳細
- **主要要素**:
  - 研究室一覧
  - 各研究室の詳細情報

### 7. 就活画面（JobsPage）
- **URL**: `/jobs`
- **説明**: 就活情報の一覧と詳細
- **主要要素**:
  - 就活体験談一覧
  - 企業・選考情報

### 8. バッジ画面（BadgePage）
- **URL**: `/badges`
- **説明**: 獲得したバッジの一覧（開発中）
- **主要要素**:
  - バッジ一覧
  - 獲得条件の説明

### 9. プロフィール画面（ProfilePage）
- **URL**: `/profile`
- **説明**: 自分のプロフィール情報
- **主要要素**:
  - ユーザー情報（ユーザー名、メールアドレス）
  - 自分の投稿一覧
  - 自分の出品一覧
  - ログアウトボタン

### 10. ログイン画面（LoginPage）
- **URL**: `/login`
- **説明**: ユーザーログイン
- **主要要素**:
  - メールアドレス入力欄
  - パスワード入力欄
  - 「ログイン」ボタン
  - 「新規登録」へのリンク

### 11. 新規登録画面（RegisterPage）
- **URL**: `/register`
- **説明**: 新規ユーザー登録
- **主要要素**:
  - ユーザー名入力欄
  - メールアドレス入力欄
  - パスワード入力欄
  - パスワード確認入力欄
  - 「登録」ボタン
  - 「ログイン」へのリンク

### 12. チャット画面（ChatPage）
- **URL**: `/chat` または `/chat/:type/:id`
- **説明**: ユーザー間のメッセージング（開発中）
- **主要要素**:
  - チャット一覧
  - メッセージ入力欄
  - 送信ボタン

## ユーザーフロー

### 新規ユーザーの場合
1. **ホーム画面**でアプリの概要を確認
2. **新規登録画面**でアカウントを作成
3. **ログイン画面**でログイン
4. **ホーム画面**または**経験談一覧画面**で投稿を閲覧
5. **投稿作成画面**で自分の経験を投稿
6. **フリマ一覧画面**で出品アイテムを閲覧または出品

### 既存ユーザーの場合
1. **ログイン画面**でログイン
2. **ホーム画面**で新着・人気投稿を確認
3. **経験談一覧画面**で興味のある投稿を検索・閲覧
4. **フリマ一覧画面**で欲しいアイテムを探す
5. **投稿作成画面**で新しい投稿を作成
6. **プロフィール画面**で自分の投稿・出品を管理

### 出品アイテムの編集・削除
1. **フリマ一覧画面**で自分の出品を確認
2. **出品編集画面**でアイテム情報を編集
3. **保存**ボタンで変更を保存
4. または**削除**ボタンでアイテムを削除

## 使用方法

### ログイン方法
1. ブラウザでアプリにアクセス
2. 右上の「ログイン」ボタンをクリック
3. メールアドレスとパスワードを入力
4. 「ログイン」ボタンをクリック

### 投稿の作成方法（経験談）
1. ログイン後、右上の「投稿する」ボタンをクリック
2. 投稿タイプで「経験談」を選択
3. タイトル、本文、タグ、カテゴリを入力
4. 「保存」ボタンをクリック

### 出品の作成方法
1. ログイン後、右上の「投稿する」ボタンをクリック
2. 投稿タイプで「出品」を選択
3. タイトル、説明、価格、画像、カテゴリを入力
4. 「保存」ボタンをクリック

### 出品の編集方法
1. **フリマ一覧画面**（`/items`）にアクセス
2. 自分の出品アイテムの「編集」ボタンをクリック
3. 編集画面で情報を変更
4. 「保存」ボタンをクリック

### 出品の削除方法
1. **出品編集画面**で「削除」ボタンをクリック
2. 確認ダイアログで「OK」をクリック

### 経験談の閲覧方法
1. **ホーム画面**で新着・人気投稿を確認
2. または**経験談一覧画面**（`/notes`）で全投稿を閲覧
3. 検索バーでキーワード検索
4. カテゴリやタグでフィルタリング

### フリマの閲覧方法
1. **フリマ一覧画面**（`/items`）にアクセス
2. カテゴリやステータスでフィルタリング
3. 検索バーでキーワード検索
4. アイテムカードをクリックして詳細を確認

## 技術スタック

### フロントエンド
- **React**: UI コンポーネントライブラリ
- **Vite**: ビルドツール
- **React Router**: ルーティング（HashRouter）
- **Axios**: HTTP クライアント
- **Context API**: 状態管理（認証、言語、投稿データ）

### バックエンド
- **FastAPI**: Python Web フレームワーク
- **PostgreSQL**: リレーショナルデータベース
- **SQLAlchemy**: ORM
- **MinIO**: オブジェクトストレージ（画像保存）
- **Redis**: キャッシュ・レート制限
- **JWT**: 認証トークン
- **Argon2**: パスワードハッシュ

### デプロイメント
- **フロントエンド**: GitHub Pages（静的ホスティング）
- **バックエンド**: Docker Compose（開発環境）

## 将来の拡張性

### 計画中の機能
- **リアルタイムチャット**: WebSocket によるメッセージング
- **通知システム**: いいね、コメント、メッセージの通知
- **フォロー機能**: ユーザー間のフォロー関係
- **コメント機能**: 投稿へのコメント
- **バッジシステム**: 活動に応じたバッジ獲得

### 技術的改善

- **画像最適化**: 自動リサイズ・WebP 変換

## まとめ

MemoLucky は、大学生がキャンパスライフで必要な情報を共有し、助け合えるプラットフォームです。経験談、フリマ、研究室情報、就活情報など、学生生活に役立つ機能を提供します。現在は基本的な CRUD 機能を実装し、将来的にはリアルタイムチャットや通知システムなどの高度な機能を追加する予定です。
